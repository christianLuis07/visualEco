## Mekanisme Keamanan: lockForUpdate() & DB::transaction()

### Ancaman: Double-Redeem via Race Condition

Seorang pengguna dengan saldo 50 poin bisa mengirim 2 request `POST /redeem` secara simultan (misal via Burp Suite Intruder). Tanpa penguncian, kedua request membaca `points_balance = 50` pada saat yang sama, keduanya lolos pengecekan saldo, dan pengguna berhasil menukar 2 reward padahal poinnya hanya cukup untuk 1.

### Solusi: Pessimistic Locking

```php
DB::transaction(function () {
    $user   = User::lockForUpdate()->find($id);    // SELECT ... FOR UPDATE
    $reward = Reward::lockForUpdate()->find($id);   // SELECT ... FOR UPDATE
    // Pengecekan saldo & stok di sini dijamin akurat
});
```

**`lockForUpdate()`** menerjemahkan ke SQL `SELECT ... FOR UPDATE` yang menahan row-level exclusive lock pada MySQL InnoDB. Request kedua yang datang bersamaan akan **menunggu** (blocking) sampai transaksi pertama selesai (COMMIT/ROLLBACK). Setelah lock dilepas, request kedua membaca data yang sudah diperbarui — sehingga pengecekan saldo pasti akurat.

**`DB::transaction()`** memastikan atomicity: jika salah satu operasi gagal (decrement stok, decrement saldo, insert record), seluruh perubahan di-rollback dan tidak ada data parsial yang tersimpan.

Kombinasi keduanya menjamin bahwa:
1. Tidak ada poin yang bisa didebit melebihi saldo aktual
2. Tidak ada stok reward yang bisa menjadi negatif
3. Setiap mutasi poin selalu tercatat di `point_ledgers`

## Kontrak Response API

### Endpoint

```
POST /api/v1/redeem
Header: Authorization: Bearer {token}
Body: { "reward_id": 1 }
```

### Sukses — 201 Created

```json
{
    "success": true,
    "message": "Penukaran reward berhasil.",
    "data": {
        "redeem_id": "9b1deb4d-3b7d-4bad-9bdd-2b0d7b3dcb6d",
        "reward_title": "Voucher Sembako Rp20.000",
        "points_spent": 50,
        "redemption_code": "VSEC-A7KX2B9M",
        "status": "pending",
        "points_balance": 100
    }
}
```

### Gagal — 422: Saldo Tidak Cukup

```json
{
    "success": false,
    "message": "Saldo poin Anda tidak mencukupi."
}
```

### Gagal — 422: Stok Habis

```json
{
    "success": false,
    "message": "Stok reward sudah habis."
}
```

### Gagal — 422: Reward Tidak Ditemukan

```json
{
    "success": false,
    "message": "The selected reward id is invalid.",
    "errors": {
        "reward_id": ["The selected reward id is invalid."]
    }
}
```

### Error Internal — 500

```json
{
    "success": false,
    "message": "Terjadi kesalahan internal pada server."
}
```
