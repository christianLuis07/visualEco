## Status Cakupan (Coverage Estimate)

Sembilan skenario ini mengunci **empat lapisan kritis** pipeline API Visueco:

1. **Authentication Guard** — Memastikan endpoint tidak bisa diakses tanpa token. Satu test (`rejects_unauthenticated`) menjamin middleware `auth:sanctum` aktif dan berfungsi.

2. **Validation Layer** — Tiga test (`missing`, `invalid format`, `exceeds size`) memverifikasi bahwa `ScanTrashRequest` menolak input buruk **sebelum** menyentuh service atau database. Ini adalah pertahanan pertama terhadap payload berbahaya.

3. **Business Logic & Anti-Fraud** — Dua test (`non_recyclable`, `low_confidence`) membuktikan bahwa sistem menolak objek non-sampah dan gambar ambigu. Tanpa test ini, celah fraud bisa lolos ke produksi dan pengguna bisa mengumpulkan poin dengan foto sembarang.

4. **External Service Resilience** — Dua test (`timeout`, `server_error`) memastikan kegagalan server AI tidak menjatuhkan seluruh aplikasi. `AiServiceException` ditangkap dan dikonversi menjadi respons JSON yang informatif, bukan stack trace mentah.

**Happy path** (`success_and_awards_points`) adalah tes terpenting karena memverifikasi **integritas transaksional end-to-end**: file tersimpan, `waste_scans` terisi, `point_ledgers` tercatat, dan `points_balance` di tabel `users` bertambah dalam satu transaksi atomik.

Cakupan ini memastikan bahwa ketika frontend diintegrasikan pada tahap berikutnya, backend sudah terbukti stabil, aman dari fraud, dan tahan terhadap kegagalan layanan eksternal.

## Bug Fix selama Tahap ini

### GD Extension tanpa dukungan JPEG

**Gejala:** `UploadedFile::fake()->image('botol.jpg')` melempar `LogicException: imagejpeg function is not defined` — 7 dari 9 test gagal.

**Akar Masalah:** Base image `php:8.2-fpm` menginstal GD tanpa flag `--with-jpeg`. Fungsi `imagejpeg()` tidak dikompilasi ke dalam ekstensi, sehingga Laravel `FileFactory` tidak bisa membuat fake JPEG.

**Perbaikan pada `Dockerfile`:**
1. Ditambahkan dependensi sistem `libjpeg62-turbo-dev`.
2. Ditambahkan langkah `docker-php-ext-configure gd --with-jpeg` sebelum `docker-php-ext-install`.
3. Image di-rebuild dengan `docker compose build visueco-app --no-cache`.
