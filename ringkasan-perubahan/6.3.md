## Bug #1 — "Unauthenticated" (401)

### Akar Masalah

Endpoint `POST /api/v1/scan` dilindungi `auth:sanctum`. Agar AJAX dari halaman
web (dengan cookie session) dianggap terautentikasi, Sanctum **stateful** harus
mengenali host browser. Tapi:

- `.env` punya `APP_URL=http://localhost` (tanpa `:8000`)
- `SANCTUM_STATEFUL_DOMAINS` **tidak diset** → default tidak memuat `localhost:8000`

Browser mengakses `localhost:8000`, tak cocok daftar stateful → request dianggap
**stateless** → tak ada sesi → **401**.

### Perbaikan

```env
APP_URL=http://localhost:8000
SANCTUM_STATEFUL_DOMAINS=localhost:8000,127.0.0.1:8000
```

Verifikasi `config('sanctum.stateful')` kini memuat `localhost:8000` & `127.0.0.1:8000`.

## Bug #2 — "Kesalahan Internal Server" (500)

### Akar Masalah

Log menunjukkan:

```
SQLSTATE[23000]: foreign key constraint fails
waste_scans.waste_category_id REFERENCES waste_categories(id)
... insert waste_category_id = 1 ...
```

**Ketidakcocokan kontrak ML ↔ Database:**
- `ml-service/category_map.py` mengembalikan `category_id` **1–5** (hardcoded)
- Database punya kategori ber-id **10–14** — karena `db:seed` sebelumnya pakai
  `insert()` tanpa id eksplisit, sementara auto-increment sudah lanjut dari
  sequence lama. Insert `waste_category_id=1` gagal FK → 500.

### Perbaikan

Seeder mengunci id kategori **1–5 secara eksplisit** agar selalu sinkron dengan ML:

```php
$categories = [
    ['id' => 1, 'name' => 'Plastik',  ...],
    ['id' => 2, 'name' => 'Kertas',   ...],
    ['id' => 3, 'name' => 'Logam',    ...],
    ['id' => 4, 'name' => 'Kaca',     ...],
    ['id' => 5, 'name' => 'Organik',  ...],
];
foreach ($categories as $cat) {
    WasteCategory::updateOrCreate(['id' => $cat['id']], [...]);
}
```

Data kategori lama (id 10–14) dihapus (tidak ada FK aktif yang mereferensi —
semua scan sebelumnya gagal), lalu di-seed ulang dengan id benar.

### Bonus: Seeder kini idempotent

`User`, `WasteCategory`, dan `Reward` semua diubah ke `updateOrCreate` agar
`db:seed` **aman dijalankan ulang** tanpa duplikat & tanpa perlu `migrate:fresh`.
Ini selaras dengan aturan: **jangan pernah `migrate:fresh`**.
