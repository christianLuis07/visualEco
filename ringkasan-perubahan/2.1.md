## Bug Fix selama Tahap ini

### `points_balance` tidak ada di `$fillable` User

**Gejala:** `$user->update(['points_balance' => 100])` di test diam-diam diabaikan oleh mass-assignment guard — saldo tetap 0 sehingga semua skenario yang memerlukan saldo awal gagal.

**Perbaikan:** Ditambahkan `'points_balance'` ke array `$fillable` pada `app/Models/User.php`. Ini juga memperbaiki potensi masalah di kode produksi jika ada kebutuhan untuk mengatur saldo secara eksplisit (misal: seeder, admin panel).

---

## Analisis Ketahanan Mutasi

Empat skenario test ini menjamin bahwa **tidak ada celah bypass saldo** di level kode produksi karena setiap test memverifikasi **dua arah** — respons HTTP **dan** state database:

1. **Happy path bukan sekadar cek status 201.** Test memverifikasi bahwa `points_balance` di database berkurang tepat sebanyak `points_required`, stok reward turun tepat 1, dan record `point_ledgers` bertipe `debit` tercatat dengan amount yang benar. Jika seorang developer keliru menghapus baris `decrement()`, test ini langsung gagal.

2. **Skenario gagal bukan sekadar cek status 422.** Setiap test negatif memverifikasi bahwa saldo user, stok reward, dan tabel `reward_redeems`/`point_ledgers` **TIDAK BERUBAH** setelah request ditolak. Ini menangkap bug berbahaya dimana pemotongan saldo terjadi sebelum validasi gagal (race between validation and mutation).

3. **Isolasi per-test via `RefreshDatabase`.** Setiap skenario dimulai dari state database yang bersih. Tidak ada efek samping antar test — jika skenario 1 sukses mengurangi saldo, skenario 2 tetap memulai dari saldo yang di-set sendiri.

4. **Auth guard terpisah.** Test unauthenticated memastikan middleware `auth:sanctum` aktif di route. Tanpa test ini, jika seseorang keliru menghapus middleware, endpoint terbuka untuk publik.

Kombinasi verifikasi **response + database state + auth boundary** pada setiap skenario memastikan bahwa perubahan kode apa pun yang merusak integritas finansial akan langsung terdeteksi oleh pipeline CI sebelum masuk ke branch utama.
