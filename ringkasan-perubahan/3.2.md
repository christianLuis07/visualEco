## State Machine UI

```
IDLE ──[file dipilih]──▶ PREVIEW ──[klik Analisis]──▶ LOADING ──▶ SUCCESS
                                                         │            │
                                                         ▼            ▼
                                                       ERROR     Reset input
                                                         │        & update poin
                                                         ▼
                                                   Alert merah
```

| State | Elemen Terlihat | Elemen Tersembunyi |
|-------|----------------|-------------------|
| **IDLE** | placeholder icon + teks | preview, skeleton, result |
| **PREVIEW** | gambar preview, tombol aktif | placeholder, skeleton, result |
| **LOADING** | skeleton animasi, tombol "Menganalisis..." disabled | preview, placeholder, result |
| **SUCCESS** | result card + poin terupdate | skeleton |
| **ERROR** | alert banner merah | skeleton, result |

## AJAX Integration — `fetch()` ke `POST /api/v1/scan`

```javascript
const res = await fetch('/api/v1/scan', {
    method: 'POST',
    headers: {
        'X-XSRF-TOKEN': getCookie('XSRF-TOKEN'),
        'Accept': 'application/json',
    },
    credentials: 'same-origin',
    body: formData,  // FormData with 'image' field
});
```

### Mengapa `X-XSRF-TOKEN` dan bukan `@csrf`?

Route `POST /api/v1/scan` dilindungi middleware `auth:sanctum`. Karena request berasal dari halaman web (bukan SPA terpisah), kita memanfaatkan **Sanctum Stateful Authentication** yang menggunakan session cookie. Laravel secara otomatis menyetel cookie `XSRF-TOKEN` pada setiap response. JavaScript membaca cookie ini dan mengirimnya sebagai header `X-XSRF-TOKEN` — Laravel middleware `VerifyCsrfToken` memvalidasi header ini sebagai pengganti `_token` form field.

### Konfigurasi `$middleware->statefulApi()`

Ditambahkan di `bootstrap/app.php` agar Sanctum mengenali request dari domain yang sama sebagai "stateful" dan menerapkan session + CSRF middleware pada route API.

## Keamanan Upload File — Analisis Berlapis

### Layer 1: Client-Side Validation (UX Guard)

```javascript
// MIME type check
const allowed = ['image/jpeg', 'image/png', 'image/jpg'];
if (!allowed.includes(file.type)) { ... }

// Size check
if (file.size > 4 * 1024 * 1024) { ... }
```

```html
<!-- HTML accept attribute -->
<input type="file" accept="image/jpeg,image/png,image/jpg">
```

> **Catatan:** Validasi client-side bersifat UX-only — mudah di-bypass melalui DevTools atau curl. Fungsinya memberikan feedback instan sebelum upload dikirim ke server.

### Layer 2: Server-Side Validation (Security Guard)

```php
// ScanTrashRequest.php
'image' => ['required', 'image', 'mimes:jpeg,png,jpg', 'max:4096'],
```

| Rule | Fungsi Keamanan |
|------|----------------|
| `required` | Mencegah request tanpa file |
| `image` | Memverifikasi file adalah gambar valid (cek header bytes, bukan hanya ekstensi) |
| `mimes:jpeg,png,jpg` | Whitelist MIME type yang diizinkan |
| `max:4096` | Batas ukuran 4 MB — mencegah abuse disk storage |

### Layer 3: Storage Isolation

```php
$request->file('image')->store('visueco-scans', 'public');
```

File disimpan di `storage/app/public/visueco-scans/` dengan nama acak (hash) yang digenerate Laravel — bukan nama asli file. Ini mencegah:
- **Path traversal** — penyerang tidak bisa menentukan path penyimpanan
- **Filename collision** — hash unik per file
- **Extension spoofing** — Laravel menentukan ekstensi dari MIME type aktual

### Layer 4: Anti-Fraud Business Logic

```php
// ScanController.php
if (! $prediction['is_recyclable'] || $prediction['confidence_score'] < 0.60) {
    return response()->json([...], 422);
}
```

Meskipun file lolos semua validasi teknis, sistem AI harus mengonfirmasi bahwa gambar berisi sampah yang dapat didaur ulang dengan confidence >= 60%. Ini mencegah gaming poin dengan mengunggah foto sembarang.

### Layer 5: XSS Prevention pada Hasil

```javascript
function escapeHtml(text) {
    const el = document.createElement('span');
    el.textContent = text;
    return el.innerHTML;
}
```

Semua data dari response API yang dirender ke DOM melewati `escapeHtml()` — mencegah Stored XSS jika data AI mengandung karakter HTML berbahaya. Field non-HTML seperti label dan angka disisipkan via `textContent` (bukan `innerHTML`).
