## 1. Jembatan Seed: Foto Host → Dataset ML

**Masalah:** dataset ML ada di named volume Docker, foto host tak bisa langsung masuk.

**Solusi:**
- Folder host `ml-service/seed_dataset/<n>_<nama>/` (di-mount **read-only**).
- `POST /seed` → `trainer.ingest_seed()` menyalin foto ke dataset internal.
- **Idempotent**: nama file tujuan = `seed_<sha1(isi)>` → foto sama tak digandakan.

**Alur pakai:**
```bash
# 1. taruh foto di ml-service/seed_dataset/1_plastik/, 2_kertas/, dst
# 2. impor + latih
docker exec visueco-ml curl -s -X POST http://localhost:8001/seed
docker exec visueco-ml curl -s -X POST http://localhost:8001/train
```

## 2. Trainer Jujur (anti metrik palsu)

**Masalah lama:** trainer melaporkan `accuracy: 1.0` meski overfit pada data kecil.

**Perbaikan `trainer.py`:**
- **Flag `reliable`**: `true` hanya bila 5 kategori terisi & ≥5 foto/kategori.
- **Split train/val** (20%) + **EarlyStopping** hanya saat `reliable`.
- **Augmentasi ringan** (flip horizontal, numpy) → menggandakan variasi efektif, CPU-friendly.
- **class_weight** untuk kelas tak seimbang.
- Meta baru: `train_accuracy`, `val_accuracy`, `reliable`.

**Bukti kejujuran (uji 30 foto dummy warna polos):**
```
train_accuracy: 0.48   val_accuracy: 0.0   reliable: true
```
Foto warna-polos memang tak punya fitur → val 0.0. Trainer **tidak lagi** mengklaim
1.0 palsu. Dengan foto asli, val_accuracy akan bermakna.

## 3. Cold-Start Lebih Kuat

`category_map.py` diperluas dengan label ImageNet realistis tiap kategori
(mis. Plastik: `plastic_bag`, `packet`, `shopping_bag`, `lighter`; Logam:
`cleaver`, `corkscrew`, `bottlecap`; Kaca: `pitcher`, `water_jug`, `jar`).
Mengurangi kasus "objek tidak dikenali" sebelum model dilatih.

> Tetap ditegaskan dalam komentar: keyword-map hanya **fallback**; akurasi
> sejati datang dari head terlatih pada foto asli.

---

## 4. Test Coverage Fitur Belajar

`tests/Feature/Api/v1/ScanConfirmTest.php` (5 skenario, `Http::fake` ML — hermetik):
- Konfirmasi "Ya benar" → 201, training_sample, `was_prediction_correct=true`
- Koreksi kategori beda → `was_prediction_correct=false`
- scan_id invalid → 422
- Konfirmasi dobel pada scan sama → 422
- Unauthenticated → 401

## Langkah Selanjutnya (untuk kamu)

1. Kumpulkan **10-15 foto per kategori** ke `ml-service/seed_dataset/<kategori>/`.
2. Jalankan `/seed` lalu `/train`.
3. Cek `reliable` & `val_accuracy` di `/model/info` — itu metrik jujur akurasi.

> Catatan: tanpa foto asli, akurasi sejati belum bisa dibuktikan — infrastruktur
> sudah siap, kualitas akhir bergantung jumlah & keragaman fotomu.
