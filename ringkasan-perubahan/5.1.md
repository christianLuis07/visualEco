## Track A — Machine Learning Self-Hosted

### Arsitektur

```
visueco-app (Laravel) ──HTTP POST /predict──▶ visueco-ml (FastAPI)
                                                    │
                                                    ▼
                                          MobileNetV2 (ImageNet)
                                          top-5 prediksi → keyword map
                                                    │
                                                    ▼
                                          5 kategori sampah Visueco
```

### Model & Stack

- **Base:** `python:3.11-slim`
- **Framework:** FastAPI + Uvicorn (Swagger UI di `/docs`)
- **Model:** MobileNetV2 pre-trained ImageNet, CPU-only, ~14 MB
- **Bobot di-bake ke image** saat build (`RUN python -c "...MobileNetV2(weights='imagenet')"`) → container offline-ready, startup cepat.

### Strategi Klasifikasi (ImageNet → 5 Kategori)

Ambil top-5 prediksi MobileNetV2, cari label pertama yang cocok dengan keyword map:

| category_id | Kategori | Keyword ImageNet (contoh) | is_recyclable |
|-------------|----------|---------------------------|---------------|
| 1 | Plastik | water_bottle, pop_bottle, plastic_bag | true |
| 2 | Kertas | carton, envelope, notebook, paper_towel | true |
| 3 | Logam | tin_can, beer_can, nail, frying_pan | true |
| 4 | Kaca | wine_bottle, beer_glass, goblet, vase | true |
| 5 | Organik | banana, orange, apple, broccoli | false |

**Fallback:** jika top-5 tidak cocok → `category_id: 0`, `is_recyclable: false` →
gerbang anti-fraud di `ScanController` (`!is_recyclable || confidence < 0.60`) menolaknya
secara alami. ML service tidak pernah melempar error 5xx ke user.

### Kontrak Response (`POST /predict`)

```json
{
  "detected_item": "water bottle",
  "category_id": 1,
  "category_name": "Plastik",
  "type_detail": "water bottle",
  "confidence_score": 0.87,
  "is_recyclable": true,
  "instructions": ["Kosongkan isi wadah", "..."]
}
```

### Bukti Verifikasi End-to-End

```bash
$ docker exec visueco-ml curl -s http://localhost:8001/health
{"status":"ok","model":"MobileNetV2","service":"visueco-ml"}

$ docker exec visueco-ml curl -s -X POST http://localhost:8001/predict -F "image=@/tmp/t.jpg"
{"detected_item":"screw","category_id":3,"category_name":"Logam",
 "confidence_score":0.05,"is_recyclable":true,"instructions":[...]}
HTTP 200
```

Container status: `visueco-ml ... Up (healthy)`.

## Track B — Frontend Build Fix (Node.js + Tailwind)

### Masalah

Layout Blade memakai `@vite(...)` tapi tidak ada Node.js di Docker — halaman muncul tanpa styling.
Selain itu user mengalami error PowerShell:

```
docker exec visueco-app npm install && docker exec ...
The token '&&' is not a valid statement separator in this version.
```

### Solusi

1. **Node.js 20 ditambahkan ke `Dockerfile`** visueco-app via NodeSource:
   ```dockerfile
   && curl -fsSL https://deb.nodesource.com/setup_20.x | bash - \
   && apt-get install -y --no-install-recommends nodejs \
   ```
2. **Rebuild container** → `node v20.20.2` terverifikasi.
3. **Build aset** berhasil:
   ```
   public/build/assets/app-cgKMVFlb.css  59.82 kB │ gzip: 11.99 kB
   public/build/assets/app-DMRtMKZ6.js   45.65 kB │ gzip: 17.72 kB
   ✓ built in 32.72s
   ```

### Catatan PowerShell (`&&`)

Windows PowerShell 5.1 tidak mendukung `&&`. Jalankan perintah **terpisah per baris**:

```powershell
docker exec visueco-app npm install
docker exec visueco-app npm run build
```

README sudah diperbarui dengan catatan ini.

## Mengapa Test Tetap Hermetik

Test memakai `Http::fake(['api.ai-ecosort.local/*' => ...])` yang mencocokkan **URL endpoint**.
Setelah endpoint produksi diubah ke `visueco-ml:8001`, fake tidak lagi cocok dan request "bocor",
menyebabkan 4 test gagal.

**Fix:** pin `AI_ECOSORT_ENDPOINT` di `phpunit.xml` ke URL fake lama
(`https://api.ai-ecosort.local/v1/predict`). Dengan begitu:
- **Produksi** memakai `visueco-ml` (dari `.env`)
- **Test** memakai URL fake (dari `phpunit.xml`) → tidak menyentuh container ML sungguhan

Hasil: **15 passed (89 assertions)** kembali hijau.
