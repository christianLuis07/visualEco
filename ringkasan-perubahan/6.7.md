# Tahap 6.7 — Fix: Admin Diarahkan ke Panel Admin Saat Login

**Tanggal Perubahan:** 16 Juni 2026

---

## Masalah yang Dilaporkan

> "Saat login admin, dashboard-nya sama seperti user? Aku heran."

Benar — admin login selalu melihat **dashboard warga**, padahal panel admin
sudah ada. User tak pernah diarahkan ke sana.

---

## Akar Masalah

`AuthController::login()` mengarahkan **semua** user ke `/dashboard`:
```php
return redirect()->intended('/dashboard');  // admin & warga sama-sama ke sini
```
Panel admin (`/admin`) **ada dan berfungsi penuh**, tapi tak ada yang mengarahkan
admin ke sana setelah login. Akses hanya bisa lewat ketik manual URL.

**Bukti panel admin memang ada:**
```
/admin → <title>Admin Panel — Visueco</title>
       → "Panel Administrasi" + "Status Model AI" ✓
```

---

## Daftar File yang Diubah

| Status | Path | Keterangan |
|--------|------|------------|
| **Ubah** | `app/Http/Controllers/Web/AuthController.php` | Redirect login per-peran |
| **Ubah** | `resources/views/admin/dashboard.blade.php` | Badge "Admin" + link "Mode Scan" |
| **Ubah** | `resources/views/dashboard.blade.php` | Link "Panel Admin" (hanya untuk admin) |
| **Baru** | `ringkasan-perubahan/6.7.md` | Dokumen ini |

---

## Perbaikan

### 1. Redirect berdasarkan peran
```php
$home = Auth::user()->role === 'admin' ? '/admin' : '/dashboard';
return redirect()->intended($home);
```

### 2. Navigasi dua arah
- **Panel admin** → tombol "Mode Scan" (admin tetap bisa scan sampah) + badge "Admin".
- **Dashboard scan** → tombol "Panel Admin" (muncul **hanya** bila `role === 'admin'`).

## Catatan

Warga biasa **tetap tidak bisa** akses `/admin` (dilindungi `role:admin`
middleware) — yang berubah hanya tujuan redirect & navigasi. Keamanan tak berubah.
