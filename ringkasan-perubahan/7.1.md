# Tahap 7.1 — UI Kelola Reward di Panel Admin (CRUD via Antarmuka)

**Tanggal Perubahan:** 16 Juni 2026

---

## Ringkasan

Operasi CRUD Reward (Update/Delete) sudah ada & teruji di API (Tahap 7.0), tapi
belum bisa diklik dari antarmuka. Tahap ini menambahkan **UI lengkap kelola
reward** di panel admin agar CRUD dapat didemonstrasikan langsung — menggenapi
syarat "Fitur Manajemen Data (CRUD)" UAS secara visual.

## Fitur UI

| Aksi | Cara | Endpoint dipakai |
|------|------|------------------|
| Lihat | Tabel reward (judul, poin, stok) | data dari controller |
| Tambah | Tombol "+ Tambah Reward" → modal form | POST `/api/v1/admin/rewards` |
| Edit | Tombol "Edit" per baris → modal terisi | PUT `/api/v1/admin/rewards/{id}` |
| Hapus | Tombol "Hapus" → modal konfirmasi | DELETE `/api/v1/admin/rewards/{id}` |

### Detail UX & Keamanan
- **Validasi inline**: error 422 dari server ditampilkan di bawah tiap field.
- **Anti-XSS**: baris tabel di-render via `createElement` + `textContent`,
  tidak ada `innerHTML` untuk data dinamis.
- **Konfirmasi hapus** modal terpisah (cegah klik tak sengaja).
- **Delete-guard**: reward dengan riwayat penukaran ditolak server (422) →
  pesan ditampilkan, baris tetap ada.
- Tabel diperbarui **tanpa reload** (prepend baris baru / update / remove).
- Tetap **0 inline script/style** (patuh aturan isolasi kode).
