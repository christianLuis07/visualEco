## Arsitektur Route Setelah RBAC

```
POST /api/v1/scan          ── auth:sanctum ──────────────── ScanController
POST /api/v1/redeem        ── auth:sanctum ──────────────── RedeemController
POST /api/v1/admin/rewards ── auth:sanctum ── role:admin ── AdminRewardController@store
```

Middleware dieksekusi secara bertingkat:
1. `auth:sanctum` — verifikasi token (401 jika gagal)
2. `role:admin` — verifikasi role (403 jika bukan admin)

User biasa yang mencoba mengakses `/api/v1/admin/*` akan dihentikan di layer kedua dengan respons 403 — **sebelum** kode controller dieksekusi.

## Analisis Mitigasi OWASP

### BFLA — Broken Function Level Authorization (API5:2023)

**Vektor Serangan:** Seorang user biasa (role: `user`) mengetahui URL admin endpoint (misal dari JavaScript bundle atau network tab). Ia mencoba mengirim request langsung via Postman/curl:

```bash
curl -X POST https://visueco.app/api/v1/admin/rewards \
  -H "Authorization: Bearer <user_token>" \
  -d '{"title":"Free iPhone","points_required":1,"stock":9999}'
```

**Tanpa RBAC:** Request lolos autentikasi (token valid), controller `store()` dieksekusi, dan user berhasil membuat reward palsu dengan poin murah — merusak ekonomi poin seluruh platform.

**Dengan RBAC (middleware `role:admin`):** Request dihentikan di middleware layer **sebelum** menyentuh controller. Respons yang dikembalikan:

```json
{
    "success": false,
    "message": "Akses ditolak. Anda tidak memiliki hak akses untuk fungsi ini."
}
```

Status: **403 Forbidden**. Tidak ada data yang ditulis ke database.

### BOLA — Broken Object Level Authorization (API1:2023)

Middleware `role` melindungi di level **fungsi** (function-level). Untuk perlindungan di level **objek** (object-level) — misal user A mencoba mengakses data user B — akan diterapkan melalui Policy/Gate pada tahap selanjutnya. Pada tahap ini, endpoint yang ada (`scan`, `redeem`) sudah aman dari BOLA karena selalu menggunakan `$request->user()` untuk mengidentifikasi pemilik data, bukan menerima `user_id` dari input.
