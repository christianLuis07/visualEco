## 1. CRUD Reward Lengkap

Sebelumnya `AdminRewardController` hanya `store()`. Kini CRUD penuh:

| Operasi | Method | Endpoint |
|---------|--------|----------|
| Create | POST | `/api/v1/admin/rewards` |
| Read (list) | GET | `/api/v1/admin/rewards` |
| Read (detail) | GET | `/api/v1/admin/rewards/{reward}` |
| **Update** | PUT/PATCH | `/api/v1/admin/rewards/{reward}` |
| **Delete** | DELETE | `/api/v1/admin/rewards/{reward}` |

**Keamanan:**
- Validasi ketat (tipe, panjang, rentang, judul unik kecuali diri sendiri).
- Delete **diblokir** bila reward punya riwayat penukaran (integritas data → 422).
- Semua dilindungi `role:admin` + dicatat di log keamanan.

## 2. Logging & Monitoring (OWASP A09)

Channel `security` baru → `storage/logs/security-YYYY-MM-DD.log`, terpisah dari
log aplikasi. Peristiwa tercatat:
- Login sukses/gagal, logout, register
- Akses ditolak (`authz.denied`)
- CRUD reward (created/updated/deleted)
- Voucher completed
- Reset password (requested/success/failed)

**Prinsip:** password & kredensial TIDAK PERNAH dicatat (hanya email + IP + UA).

## 3. Reset Password (Nilai Tambah)

Memakai password broker bawaan Laravel:
- Token acak (hashed di DB), kedaluwarsa 60 menit, throttle 60 detik.
- **Anti-enumeration**: pesan sukses selalu sama walau email tak terdaftar.
- Password baru di-hash; route diberi `throttle:6,1`.
- Email reset → log (`MAIL_MAILER=log`), cocok demo tanpa SMTP.
