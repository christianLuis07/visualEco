## AJAX Redemption Flow

```
Klik "Tukarkan Poin"
   │
   ▼
Modal Konfirmasi (judul + biaya poin)
   │
   ├─[Batal]──▶ Tutup modal
   │
   └─[Ya, Tukarkan]──▶ fetch POST /api/v1/redeem
                           │
                           ├─ 201 ──▶ Modal Sukses
                           │         • redemption_code (mono bold)
                           │         • Update saldo poin real-time
                           │         • Reload on close (refresh stok + ledger)
                           │
                           ├─ 422 ──▶ Alert banner merah (5 detik auto-hide)
                           │         • "Saldo poin Anda tidak mencukupi."
                           │         • "Stok reward sudah habis."
                           │
                           └─ Network error ──▶ Alert banner merah
```

### Request Format

```javascript
fetch('/api/v1/redeem', {
    method: 'POST',
    headers: {
        'X-XSRF-TOKEN': getCookie('XSRF-TOKEN'),
        'Accept': 'application/json',
        'Content-Type': 'application/json',
    },
    credentials: 'same-origin',
    body: JSON.stringify({ reward_id: Number(rewardId) }),
});
```

## Keamanan Integritas Data Visual

### 1. Perlindungan DOM-based XSS pada Kode Voucher

Semua data dinamis dari response API (`redemption_code`, `reward_title`, `points_balance`) disisipkan ke DOM menggunakan `textContent`, bukan `innerHTML`:

```javascript
document.getElementById('success-code').textContent = data.redemption_code;
document.getElementById('success-title').textContent = data.reward_title;
```

Property `textContent` secara otomatis meng-escape karakter HTML (`<`, `>`, `&`, `"`) — sehingga meskipun response API di-inject dengan payload `<script>alert(1)</script>`, browser akan menampilkannya sebagai teks biasa, bukan mengeksekusinya. Ini menutup vektor **DOM-based XSS** pada penampilan voucher secara dinamis.

### 2. Integritas Data Credit/Debit pada Tabel Ledger

Tabel riwayat menerapkan **pemisahan warna visual** untuk transparansi data:

| Tipe | Badge | Warna Jumlah | Prefix |
|------|-------|-------------|--------|
| `credit` (poin masuk) | `bg-teal-50 text-teal-700` "Masuk" | `text-teal-600` | `+` |
| `debit` (poin keluar) | `bg-amber-50 text-amber-700` "Keluar" | `text-amber-600` | `-` |

Diferensiasi warna ini memastikan warga dapat langsung membedakan transaksi masuk dan keluar tanpa perlu membaca detail — mendukung prinsip **transparansi data keuangan mikro** di tingkat komunitas.

### 3. Server-Side Data Boundary

Data ledger difilter secara ketat di route:

```php
PointLedger::where('user_id', auth()->id())->...
```

User hanya melihat riwayat miliknya sendiri — tidak ada kemungkinan enumerasi data user lain melalui manipulasi parameter. Query ini dikombinasikan dengan middleware `auth` yang memastikan hanya user terautentikasi yang dapat mengakses halaman.
